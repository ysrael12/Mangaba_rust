1//! # Mangaba AI
2//!
3//! A modular, provider-agnostic LLM agent framework in Rust.
4//!
5//! ## Modules
6//!
7//! | Module | Description |
8//! |--------|-------------|
9//! | [`core::agent`] | Agent with memory, guardrails, tools, delegation, and ReAct execution |
10//! | [`core::callbacks`] | Hook system for step/tool/LLM/task lifecycle events |
11//! | [`core::config`] | Environment-based provider detection and [`LLMConfig`](core::types::LLMConfig) builder |
12//! | [`core::crew`] | Multi-agent orchestration (sequential / hierarchical) |
13//! | [`core::embeddings`] | Embedding trait + OpenAI, HuggingFace, NoOp, Cached LRU, vector store |
14//! | [`core::errors`] | [`MangabaError`](core::errors::MangabaError) enum (thiserror, 18 variants, retryable detection) |
15//! | [`core::events`] | Global EventBus for subscribe/emit decoupled communication |
16//! | [`core::guardrails`] | Length, profanity, composite guardrails + [`GuardrailTool`](core::guardrails::GuardrailTool) |
17//! | [`core::llm`] | [`LLMClient`](core::llm::LLMClient) trait + 7 providers + streaming + retry + cache + token counter |
18//! | [`core::memory`] | Short-term, long-term (JSON-persisted), and entity memory stores |
19//! | [`core::output_parsers`] | [`JSONOutputParser`](core::output_parsers::JSONOutputParser), [`NoOpOutputParser`](core::output_parsers::NoOpOutputParser) |
20//! | [`core::pipeline`] | Stage, ParallelStage, ConditionalStage, Pipeline for workﬂow composition |
21//! | [`core::planner`] | [`PlanStep`](core::planner::PlanStep), [`ExecutionPlan`](core::planner::ExecutionPlan), [`TaskPlanner`](core::planner::TaskPlanner) (LLM-generated plans) |
22//! | [`core::prompt_templates`] | [`PromptTemplate`](core::prompt_templates::PromptTemplate), [`SystemPromptBuilder`](core::prompt_templates::SystemPromptBuilder) |
23//! | [`core::protocols`] | A2A (agent-to-agent) + MCP (model context protocol) |
24//! | [`core::rag`] | RAG engine: ingest files/text → chunk → embed → query |
25//! | [`core::react`] | [`ReActEngine`](core::react::ReActEngine): Thought → Action → Observation loop |
26//! | [`core::retry`] | Exponential backoff with jitter for LLM calls |
27//! | [`core::task`] | Task with context chaining and output persistence |
28//! | [`core::tools`] | [`BaseTool`](core::tools::BaseTool) trait + Calculator, File, Text, Web search tools |
29//! | [`core::types`] | Core data types: [`LLMConfig`](core::types::LLMConfig), [`Message`](core::types::Message), [`ToolCall`](core::types::ToolCall), etc. |
